# TKA環境向け検証スクリプト (verify_tka.py) の使い方

このスクリプトは、Min-Camlコンパイラ (`TKA` バックエンド) の動作検証を自動化するためのツールです。
コンパイル、アセンブル、シミュレーション実行を行い、その結果（標準出力）をOCamlインタプリタの正解と比較検証します。

## 概要
- **対象**: `test/` ディレクトリ内の `.ml` ファイル
- **機能**:
  1. `min-caml` コンパイラの実行（`.s` 生成 + `TKA/libmincaml.S` の自動結合）
  2. `ocaml` インタプリタによる正解データの生成（`.ans`）
  3. （オプション）アセンブラによるバイナリ生成（`.bin`）
  4. （オプション）シミュレータによる実行と数字列の比較（浮動小数点の誤差許容）

## 前提条件
- Python 3
- OCaml環境（`dune`, `ocaml` コマンド）
- Min-Camlのビルド済みバイナリ (`min-caml` または `_build/default/main.exe`)
- (シミュレーション用) 外部シミュレータとアセンブラ（例: `bill_sim`）

## 基本的な使い方（コンパイル確認のみ）

シミュレータがない場合でも、コンパイルの成功可否を確認できます。

```bash
# 事前にビルド
dune build

# 全テストのコンパイル確認
./verify_tka.py
```

## シミュレータを用いた検証 (bill_sim の例)

デフォルト設定として `bill_sim` 用の引数がハードコードされているため、特別な引数指定なしで実行可能です。

```bash
# 全テスト実行
./verify_tka.py

# 特定のテストのみ実行
./verify_tka.py --test ack
```

もちろん、従来のオプション指定による上書きも可能です。

### Clean 機能 (生成ファイルの削除)

`test/` ディレクトリ内の生成ファイル（`.s`, `.ans`, `.bin` など、`.ml` 以外のファイル）を一括削除するには `--clean` を使用します。

```bash
./verify_tka.py --clean
```

### オプション詳細

| オプション | 説明 | デフォルト値 |
|------------|------|----|
| `--test NAME` | 指定したテストのみを実行します（拡張子 .ml は不要） | (全テスト実行) |
| `--clean` | `test/` 以下の生成ファイルを削除して終了します | - |
| `--sim CMD` | シミュレータを実行するコマンド。`{}` は入力ファイルに置換されます。 | `'../bill_sim/sim {} --headless'` |
| `--asm CMD` | アセンブラコマンド。`.s` から `.bin` を生成します。`tkasm <src> -o <dst>` 形式を想定しています。 | `'../bill_sim/tkasm'` |
| `--trim-after STR` | シミュレータ出力のうち、この文字列以降を無視します。 | `"Headless finished"` |

## 検証の詳細
- **ライブラリリンク**: コンパイル後、自動的に `TKA/libmincaml.S` がアセンブリファイルの末尾に結合されます。これにより `min_caml_print_char` 等のIO関数が利用可能になります。
- **浮動小数点の誤差**:
    - **1e-6以下**: 誤差なしとして `PASS`
    - **1e-3以下**: 許容範囲内として `PASS (with warnings)`
    - **それ以上**: `FAIL`

## トラブルシューティング
- **COMPILE FAILED**: `min-caml` の生成に失敗しているか、ライブラリ (`Lib/library.ml`) のパースエラーの可能性があります。
- **ASM FAILED**: アセンブラのパスが間違っているか、命令セットの不一致があります。
- **SIM FAILED / Mismatch**: 
    - "got 0" (出力なし): ライブラリが正しくリンクされていないか、サイクルオーバーでプログラムが途中終了している可能性があります。
    - "Mismatch": OCamlの期待値と一致しませんでした。出力ログを確認してください。
