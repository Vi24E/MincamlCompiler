# bill_sim - CPUシミュレータ

カスタムCPUアーキテクチャ（TKMEDA）用のシミュレータです。

## ビルド & 実行

```bash
# ビルド
make

# アセンブル (.s → .bin)
./tkasm hoge.s -o hoge.bin
# 注意: finラベルの無限ループパターンは自動的に quit 命令に変換されます。

# シミュレータ起動
./sim hoge.bin          # TUIモード
./sim hoge.bin --cli    # CLIモード
./sim hoge.bin --headless  # ヘッドレス（自動実行）
```

## 動作モード

| モード | 説明 |
|--------|------|
| **TUI** | ncursesベースのグラフィカルUI。コード・レジスタ・メモリを表示 |
| **CLI** | 対話的デバッガ |
| **Headless** | 最大1000万サイクル自動実行 |

## TUI操作キー

| キー | 機能 |
|------|------|
| `s` | 1ステップ実行 |
| `r` | 連続実行開始/停止 |
| `u` | Undo（1ステップ戻る）|
| `d` | Output を `output.bin` に保存 |
| `b` | ブレークポイント設定 |
| `q` | 終了 |

## CLIコマンド

| コマンド | 機能 |
|----------|------|
| `s` | 1ステップ |
| `n <N>` | Nステップ実行 |
| `r` | 連続実行（上限10億サイクル）。`quit` 命令到達で停止。 |
| `o` | Step Out（関数リターンまで）|
| `p` | レジスタおよび出力バッファの内容を表示 |
| `a` | PC + レジスタ全表示 |
| `m <addr> [count]` | メモリダンプ |
| `q` | 終了 |

## 仕様詳細

### 終了処理 (`quit`)
アセンブラ `tkasm` は、以下の「`fin` ラベルでの無限ループ」パターンを検出すると、自動的に `quit` 命令（オペコード `0xFF`）に置き換えます。

```assembly
fin:
    set_label %i31, fin
    jmp %i31, 0(%i31)
```

シミュレータは `quit` 命令を実行すると `Program finished` と表示して停止します。

### 入力 (`input.bin`)
シミュレータは以下の順序で入力ファイルを探します：
1. 現在のカレントディレクトリ (`./input.bin`)
2. シミュレータ実行ファイルのあるディレクトリ (`dirname(argv[0])/input.bin`)

### 出力
CLIモードで `p` コマンドを実行すると、I/Oアドレス（`0xf0000000`）への書き込み内容（`output_buffer`）をASCII文字列として表示します。
```
(sim) p
...
output: Hello World
```

### レジスタ
- **整数レジスタ**: `x0` - `x31` （`x0`は常に0）
- **浮動小数点レジスタ**: `f0` - `f31`

### メモリ
- **サイズ**: 128MB（フラット）
- **I/Oアドレス**: `0xf0000000`（読み書きで標準入出力）

### 対応命令

| カテゴリ | 命令 |
|----------|------|
| 算術 | `add`, `sub`, `mul`, `div`, `rem` (+ 即値版 `-i`) |
| シフト | `sar`, `sll`, `slr` |
| 論理 | `or`, `xor` |
| アドレス計算 | `add4` (`rd <- rs1 + rs2 * 4`) |
| 比較 | `clt`, `cleq`, `ceq` |
| 分岐 | `jeq`, `jlt`, `jleq`, `jmp` |
| ロード/ストア | `lb`, `sb`, `lw`, `sw`, `lf`, `sf` |
| 即値ロード | `movi`, `movui` |
| 浮動小数点 | `fadd`, `fsub`, `fmul`, `fdiv`, `fneg`, `finv`, `fsqrt`, `frsqrt`, `fabs`, `ffloor` |
| 型変換 | `mif`, `mfi`, `ftoi`, `itof` |
| 浮動小数点比較 | `feq`, `fleq`, `flt` |
| 特殊制御 | `quit` (シミュレータ終了), `nop`, `exit` |

## その他機能

- **Undo機能**: ステップ実行時にレジスタ・メモリ変更を記録し、戻せる
- **プロファイリング**: 命令数、分岐成功率を記録
- **Step Out**: `jmp %x3`（リターン命令）検出で関数から抜ける
